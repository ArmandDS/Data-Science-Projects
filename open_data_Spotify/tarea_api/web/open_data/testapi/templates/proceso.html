Proceso de Selección/Adquisición de los datos


Objectivo: Se pretende seleccionar los datos del API de Spotify a fin de Estudiar las caracteristicas de las canciones más escuchadas a tráves de los años desde el Año 1940 hasta el 2016 inclusive, para ello seleccionaremos las canciones de la lista billboard que se publica anualmente,

Debemos hacer un scrapy de la web billboard a fin de obtener el par "<"canción, cantante">" para luego valiendonos del API de Spotify obtener las métricas respectivas

Examinamos la estructura de la página a tráves de su codigo html a fin de construir un scrapper en python, destacando lo siguiente:

  • La excepción es el año 1940 cuya estructura y url es distinto se accede con "http://billboardtop100of.com/336-2/"

El API de Spotify:

La documentacion de esta API se encuentra en la página oficial, pero basicamente se trata de establecer una conexión previo registro en la pagina de spotify y solicitar las claves/token de acceso para nuestro script

Para nuestro análisis previo registro y conexión en spotify, leeremos del archivo generado por el scrapper, iteramos sobre cada fila extrayendo el nombre del cantante y el título de la canción

En cada iteración hacemos a tráves del API una consulta por nombre y titulo para obtener las siguientes métricas :

Construcción de los grupos

El objetivo es implementar una técnica de cluster conocidad como Affinity Propagation la cuál según wikipedia es un algoritmo de cluster que no requiere una estimación a priori del número de cluster, luego construiremos un Grafo con d3.js para obtener información visual, El proceso es el siguiente:

  • Primero se carga el Archivo generado por el script de conexión al API
  • Luego Iteramos por cada década desde 1940, ¿Por qué en décadas?: pues por un tema de tiempo de computo/costo computacional
  • Por cada Década Seleccionamos las métricas, Obtenemos la traspuesta y luego la matriz de covarianza
  • A partir de la matriz de Covarianza obtenemos la Matriz de correlaciones
  • Calculamos luego la matrix de precision, entendida como la inversa de la matriz de covarianza, que será el input de la función affinity_propagation de la librería sklearn para obtener los clusters y la etiquetas de cada canción (la canciones serán los nodo en nuestro grafo)
  • Para Generar las relaciones entre las canciones utilizamos las correlaciones parciales (Serán los links de nuestro grafo)
  • Por cada década generamos un archivo json, con los nodos y las relaciones entre estos, cada archivo tiene por nombre "data"+año+".json", esto se hace con el fin de utlizar desde d3.js el Force layout
  • Por cada década también generamos un .csv que contiene el promedio de las métricas por cado grupo presente en esa década

La visualización

Elegimos la Visualizacion del tipo grafo porque consideramos que es adecuada para analizar estructuras de datos que puedan estar muy relacionadas (visualización de comunidades)como el caso de la música y las canciones que representan un reto por ser la música de naturaleza subjetiva y donde los generos musicales constantemente se solapan

En nuestro caso utlizaremos javascript y d3.js para lograr la visualización, todo esto montado sobre el framework django de python

El funcionamiento es el siguiente:

  • Al entrar en la pagina inicial se carga el archivo del año 1940(por defecto) para presentar un primer grafo
  • Cada Node Representa un track, y cada linea representa la relacion ese nodo con otros
  • Los colores representan el cluster al que pertenece la canción según el algoritmo implementado
  • El Gráfico de Barras, esta hecho de manera informativa, para comparar las métricas entre el cluster y la canción seleccionada
  • Para Seleccionar un nodo, solo se debe hacer click en el mismo, inmediatamente se envia un peticion vía ajax al servidor django que devolverá la métrica de esa canción y la metrica del grupo
  • También se envia un petición ajax cambiar de decada
  • Por último también se hace una busqueda en youtube con el fin de intentar apreciar el sonido asociado a la métrica
